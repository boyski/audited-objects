AO CHANGE LOG

1.0.10 (Nov 9, 2011):

  • Moved Java server into Git repository.

1.0.9 (Oct 1, 2011):

  • Cleanup pass over documentation.
  • Switched to use of libgit2 for SHA-1 calculations, retired OpenSSL/libcrypto.
  • Updated jar files in server.
  • Work around Ubuntu $LIB bug by using $LD_LIBRARY_PATH to find auditor on Linux.

1.0.8 (July 5, 2011):

  • This is a fairly minor release, with a couple of smaller bugfixes and some cleanup and streamlining of code.
  • Added an aoquery script for digging details out of an audit file manually.
  • Brought back Mac OS X support. The Mac binaries are "fat" in that they contain both 32- and 64-bit code (for Intel only - no PPC support).

1.0.7 (March 10, 2011):

  • Major textual simplification of client code: Windows-style "tchar" macros, which are designed to allow code to be built for single-byte or Unicode multi-byte, character sets, were removed. There is no likelihood of the client ever needing to be built for Unicode. Meanwhile this feature had not been exercised for years and was certainly broken, and of course it was non-obvious to Unix people and a pain to keep up. So, removed. Not a functional change, just a massive text edit.
  • Major bugfix for Linux. Previous versions worked on 32-bit or 64-bit Linux but were unable to function in an environment of mixed 32- and 64-bit processes, which is increasingly a common situation. This is also known as a "multilib" installation. Anyway, I finally found out how to support multilib and implemented it in this release. Starting from 1.0.7, you can choose the 32- or 64-bit binary package but each package contains both 32- and 64-bit auditors; the only difference between them is the bit width of the "ao" driver program.

1.0.6:

  • Updated third-party jars (H2).
  • Finished moving client code into Git.

1.0.5:

  • Version 1.0.5 contains a major reworking of the auto-dependency generation code. It required changing the -M and -F flags non-compatibly but I doubt anyone was using them the old way since they weren't too useful. In the new model, simply adding -M (a la gcc) will cause AO to create "dependency fragments", one per target, with a .d extension. Adding a -F foo.mk flag will cause AO to also generate a wrapper makefile which includes all those fragments. Thus, you can either take responsibility for including the fragments directly in your own Makefiles, or you can create the wrapper file and include just that, possibly by using the GNU Make MAKEFILES environment variable. The net result is that "MAKEFILES=`pwd`/foo.mk ao -M -F foo.mk make" should both create and use a set of generated dependency files without any modification of the existing Makefiles. There are other ways; see the HOWTO and Tour documents.
  • The Unix auditor library has been renamed from libao.so to libAO.so. This was done partly for consistency with the Windows client and partly to avoid possible confusion with a different "libao" library (http://www.xiph.org/ao/), an audio library which was once dead but then resuscitated. The old libao.so file shouldn't break anything but could cause confusion in future so it's best to remove it manually.

1.0.4:

  • Removed the Solaris/sparc binary. This can easily be reconstituted if there's a call for it but for now we will save the disk space and build time and concentrate on more modern platforms.
  • Now building the Linux 64-bit binary on an older system for better compatibility. Both 32- and 64-bit Linux versions now require glibc 2.3.2 or better.
  • Updated some third-party jars to latest (EclipseLink, H2, log4j, etc).
  • Made some minor recycling optimizations.

1.0.3:

  • Improved aggregation properties to better handle different compiler installation styles, particularly for g++ and specifically to fix a problem exposed by Ubuntu packaging.

1.0.2:

  • Added a new client binary for 64-bit Linux.
  • Fixed some Linux incompatibilities with Ubuntu variants.
  • Changed default project name strategy to remove any trailing "-version".
  • Reworked clients to link with libcrypto. This was done to make it structurally easier to experiment with alternate identity hash algorithms (e.g. SHA1) and also to rely on more standard, and more regularly maintained, libraries. Removed the older individual libraries like libsha2 and libvmac.
  • Upgraded the embedded DB to H2 1.2.141 (from 1.2.140).

1.0.1:

  • Moved logfiles to a subdir called "_LOGFILES_"
  • Persist command lines to a CLOB instead of a VARCHAR to avoid overflows.
  • Upgraded embedded DB to H2 1.2.140 (from 1.2.139).
