AO AND INTEGRATED DEVELOPMENT ENVIRONMENTS

AO is easily added to a traditional command-line build by simply prepending "ao run" to the command line. But what about Integrated Development Environments (IDEs) with a graphical Build button? Depending on the IDE there are a few possible answers.

TWEAK CONFIG FILE

Some IDEs may have a config file or registry entry which specifies the command to run when the Build is pressed. These should be easy; change it to prepend the ao command. There's no need to add flags here; since all options to ao simply set property values, any ao configuration can be done via property files.

IDE MODE

However, some IDEs may not make such a setting available. For these, there is an experimental, speculative IDE mode. Here, the entire IDE is run under ao control and a special property is set which specifies a regular expression. AO inhabits the IDE like a beneficial virus, doing nothing but monitoring sub-process creation. When the IDE spawns a command which matches the regular expression, however, AO springs into action and audits just that process subtree. This work is experimental and undocumented; contact us if you'd like to help test it.

S.O.L. MODE

Unfortunately, some IDEs may start the build without an accompanying new process. For instance, the make algorithm is simple enough and well enough known that the IDE may choose to inline its own dependency tree walker. Compile and link operations will almost always be in a separate process space but the control process may be within the IDE. This situation has not yet been addressed; if and when it is encountered we may look into a solution.

SUMMARY

No IDE has yet been tested with AO, and each IDE is different. The above are ideas which may pay off with certain IDEs. There is little doubt that some IDEs will be a challenge to integrate, or that others will expose AO bugs. Improvement is ongoing.
