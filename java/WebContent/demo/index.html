Users new to AO should always start with the Demo Package. Note that this is not part of the AO tool proper and is provided for test and demo purposes only. Warning: line endings on a few files, in particular .bat scripts, may need to be adjusted for Windows.

The"hello" subdirectory of the demo package contains a simple"Hello World" program. This is the simplest demo and generally should be tried first. It is what's used in the Tour of AO document. It's provided with both a Makefile and a build.sh script for Unix build purposes, as well as a NMakefile (e.g."nmake -f NMakefile") and a build.bat script for Windows. Finally, there's a Java version which may be built with Ant. Any of these build tools may be used with AO; in fact that's the point of the HelloWorld demo.

Open-source applications are an ideal way to exercise AO, and I use many for testing. Here is a partial list of some open source packages I've used AO with. These would be good candidates with which to try out AO.

PCRE Perl Compatible Regular Expressions
Jikes Jikes Java Compiler
HSQLDB 100% Java Database
GNU Make GNU Make
Bash The BASH Shell
GNU Core Utils GNU Core Utils

Each of these will have its own configuration instructions but for Unix most are of the common "./configure; make" variety. Building for Windows is usually more complex; see individual package documentation.

Note that open source software is a particularly hostile environment for AO, for reasons elaborated below. For us this makes it an ideal test platform, and for you it means that AO will generally perform better with your own builds than it does with open source. This conclusion may seem a bit surprising, so supporting arguments are provided. There's also a document on tuning a build for AO which covers related topics.
